================================================================================
GEMINI AI PROMPT-RESPONSE BENCHMARK - PROMPT TEMPLATES
================================================================================

This file contains standardized prompt templates used across the benchmark
dataset. Templates use the following placeholder format:
  {variable} - Required variable that must be filled in
  [variable] - Optional variable with a default value

================================================================================
SECTION 1: EXPLANATION TEMPLATES
================================================================================

Template ID: EXPLAIN-001
Difficulty: easy
Category: General Knowledge
---
Explain {concept} in simple terms that a {audience_level} could understand.
---

Template ID: EXPLAIN-002
Difficulty: medium
Category: Technical
---
Explain {concept} and discuss its implications for {domain}. Include at least
{min_examples} real-world examples.
---

Template ID: EXPLAIN-003
Difficulty: hard
Category: Academic
---
Provide a rigorous explanation of {concept}, including its mathematical foundation,
historical development, and modern applications in {field}. Address common
misconceptions and current research directions.
---

SECTION 2: PROGRAMMING TEMPLATES
================================================================================

Template ID: CODE-001
Difficulty: easy
Category: Programming
---

the approach.
---

Template ID: CODE-002
Difficulty: medium
Category: Programming
---
Implement {algorithm_or_data_structure} in {language} with the following features:
- {feature_1}
- {feature_2}
- {feature_3}

Include unit tests and time complexity analysis.
---

Template ID: CODE-003
Difficulty: hard
Category: Programming
---
Design and implement a {system_type} in {language} that meets the following
requirements:
- {requirement_1}
- {requirement_2}
- {requirement_3}
- {requirement_4}

The implementation should handle edge cases, include comprehensive error handling,
and be production-ready. Include documentation and examples.
---

Template ID: CODE-004
Difficulty: hard
Category: Database
---
Design a database schema for {application_type} that supports:
- {feature_1}
- {feature_2}
- {feature_3}

Requirements:
- Scale: {expected_scale}
- Query performance: {performance_requirement}
- Consistency model: {consistency_requirement}

Include table definitions, indexes, and optimization notes.
---

TEMPLATE USAGE NOTES
================================================================================

1. Variables in {braces} are MANDATORY and must be replaced with actual values.
2. Variables in [braces] are OPTIONAL and may be omitted for default behavior.
3. Each template has been tested for consistency and clarity across multiple uses.
4. Templates are designed to be language-agnostic unless a specific language is
   specified in the variable values.
5. When adding new templates, follow the existing naming convention:
   CATEGORY-NUMBER (e.g., EXPLAIN-001, CODE-002).
